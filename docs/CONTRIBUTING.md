# Contributing Guidelines

## Purpose

This document defines the engineering workflow for contributing to Omnora Student AI.

Every contribution should follow the same structure to keep the project clean, maintainable, and scalable.

---

# Branch Strategy

main

Production-ready code only.

architecture-v2

Architecture, documentation, and major structural improvements.

feature/*

New features.

bugfix/*

Bug fixes.

hotfix/*

Emergency production fixes.

---

# Commit Rules

Each commit must represent one logical change.

Good examples

docs: add security guidelines

feat: implement Navigation V2

fix: resolve sidebar scrolling issue

style: improve button spacing

refactor: simplify authentication module

Avoid mixing multiple unrelated changes in one commit.

---

# Pull Requests

Every pull request should

- Have a clear title.

- Explain what changed.

- Reference related issues if available.

- Pass testing before merging.

---

# Code Review

Before merging

- Check HTML structure.

- Check CSS consistency.

- Check JavaScript behavior.

- Check responsive design.

- Check accessibility.

- Check performance.

---

# Documentation

Major architectural changes must update documentation.

Never introduce large structural changes without updating docs.
